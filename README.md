## Project Overview

A marketing and lead-generation website for Dr. Lizzie Bronte, executive advisor and keynote speaker. The site presents her services, speaking offer, and personal story — and allows prospective clients to book a complimentary 20-minute Clarity Call directly, without any third-party scheduling tool.

### Pages

| URL | File | Purpose |
|-----|------|---------|
| `/` | `index.html` | Home — full marketing page |
| `/about` | `about.html` | Dr. Bronte's story and credentials |
| `/work-with-me` | `work-with-me.html` | The Authority Advisory program details and pricing |
| `/speaking` | `speaking.html` | Speaking and workshop offer |
| `/contact` | `contact.html` | General inquiry form |
| `/book` | `book.html` | Self-serve Clarity Call booking calendar |
| `/admin` | `admin/index.html` | Decap CMS (content editing — password protected) |

## Tech Stack

| Layer | Technology | Notes |
|-------|-----------|-------|
| Static site generator | [Eleventy (11ty) v2](https://www.11ty.dev) | LiquidJS templating |
| Hosting & CDN | [Netlify](https://netlify.com) | Auto-deploys from `netlify` branch |
| Serverless functions | Netlify Functions (Node.js 20) | `/netlify/functions/` |
| Booking storage | [Netlify Blobs](https://docs.netlify.com/blobs/overview/) | Built-in KV store, no database needed |
| Transactional email | [Resend](https://resend.com) | Booking confirmations and notifications |
| CMS | [Decap CMS](https://decapcms.org) | Git-based, edits committed to repo |
| CMS auth | Netlify Identity + Git Gateway | Managed in Netlify dashboard |
| CSS | Plain CSS with custom properties | No framework |
| JavaScript | Vanilla JS | No frontend framework |
| Fonts | Google Fonts — Cormorant Garamond (display) + DM Sans (body) | Loaded in `_layouts/default.html` |
| Package manager | npm | `package-lock.json` committed |

**Total hosting cost: $0/month.** Only paid dependency is Resend (free up to 3,000 emails/month).

## Local Development

### Prerequisites
- Node.js 20+
- npm

### Setup

```bash
# Clone the repo and switch to the active branch
git clone https://github.com/vibecodeta4/lizzie-bronte.git
cd lizzie-bronte
git checkout netlify

# Install dependencies
npm install

# Start local dev server (hot-reload)
npm run serve
# → Site available at http://localhost:8080
```

> **Note:** The booking calendar (`/book`) calls `/api/slots` and `/api/book` which are Netlify Functions. These will not work locally without the [Netlify CLI](https://docs.netlify.com/cli/get-started/). For local function testing run `netlify dev` instead of `npm run serve`.

## Deployment

Netlify is connected to the **`netlify` branch** of the GitHub repo. Every push to that branch triggers an automatic redeploy.

```bash
# Make changes, then deploy
git add .
git commit -m "Description of change"
git push origin netlify
```

Netlify runs:
```
npm ci && npm run build
```
…and publishes the `_site/` directory.

**Build time:** ~30–60 seconds.

## Environment Variables

Set in Netlify dashboard → **Site Configuration → Environment Variables**.  
Never commit these to the repository.

| Variable | Required | Description |
|----------|----------|-------------|
| `RESEND_API_KEY` | Yes (for emails) | API key from [resend.com](https://resend.com). Free account allows 3,000 emails/month. |
| `BOOKING_NOTIFICATION_EMAIL` | Yes (for emails) | Email address that receives new booking alerts (Dr. Bronte's inbox). |
| `RESEND_FROM_EMAIL` | Optional | Sender address shown in emails. Defaults to `Bronte Advisory <bookings@bronteadvisory.com>`. Must be a verified domain in Resend. |

### Setting up Resend (one-time, ~10 minutes)

1. Go to [resend.com](https://resend.com) → create a free account
2. **Domains** → Add your domain → follow DNS verification steps
3. **API Keys** → Create Key → copy the value
4. In Netlify dashboard → Environment Variables → add `RESEND_API_KEY`
5. Add `BOOKING_NOTIFICATION_EMAIL` with Dr. Bronte's email
6. Trigger a redeploy (push any small change or use Netlify dashboard → Deploys → Trigger deploy)

## Booking System

The self-serve booking calendar at `/book` is built entirely in-house — no Calendly or third-party scheduler.

### How it works

1. **Visitor loads `/book`** → JavaScript fetches `GET /api/slots`
2. **`get-slots.js`** reads `_data/availability.yml` to generate all possible slots for the next 28 days, queries Netlify Blobs to find already-booked slots, and returns only the free ones
3. **Visitor selects a slot and submits the form** → JavaScript posts to `POST /api/book`
4. **`create-booking.js`** checks the slot isn't already taken, saves it to Netlify Blobs, sends a notification email to Dr. Bronte, and sends a confirmation email to the visitor
5. **Success screen** is shown to the visitor

### Changing availability

Dr. Bronte's available days and hours are controlled entirely in the CMS (**Settings → Booking Availability**) — no code changes needed.

Fields:
- **Timezone** — e.g. `America/New_York`
- **Available days** — checkboxes Mon–Sun
- **Available from / until** — e.g. `09:00` to `17:00`
- **Call duration** — default 20 minutes
- **Buffer between calls** — default 10 minutes
- **Booking window** — how many days ahead visitors can book (default 28)

## CMS Usage Guide

The CMS is at **https://drlizziebronte.netlify.app/admin**

Log in with the email and password set up via Netlify Identity.

### What can be edited in the CMS

#### Testimonials
Add, edit, or remove client testimonials shown on the home page.
- **Featured Testimonial** — displayed as the large pull-quote on the home page
- **Testimonial Cards** — the smaller cards in the testimonials grid

#### Pricing
Update program investment amounts without touching code.
- **Monthly Amount** — e.g. `$1,250`
- **Monthly Period Label** — e.g. `per month × 3`
- **Full Payment Amount** — e.g. `$2,999`
- **Full Payment Label** — e.g. `paid in full`
- **Program Duration** — e.g. `12-Week Private 1:1 Program`

#### Settings
- **LinkedIn URL** — full URL to Dr. Bronte's LinkedIn profile
- **Instagram URL** — full URL to Instagram
- **Contact Email** — displayed in the footer
- **Clarity Call Spaces Note** — short note shown on the contact page, e.g. `Limited spaces available`

#### Booking Availability
Control when clients can book Clarity Calls (see [Booking System](#booking-system) above).

#### Articles
Create and publish blog posts / articles shown on the home page and a future articles page.
- Fill in Title, Publish Date, Tag, Excerpt, Featured Image, and Body (rich text)
- Posts are saved as Markdown files in `_posts/`

### Publishing changes

The CMS commits directly to the `netlify` branch in GitHub. Netlify detects the new commit and redeploys automatically — changes are live within ~60 seconds of saving in the CMS.

## Ongoing Maintenance

### Adding a new page
1. Create `pagename.html` at the repo root with the Eleventy front matter:
   ```yaml
   ---
   layout: default
   title: "Page Title — Bronte Advisory"
   description: "Meta description"
   page_css: css/pagename.css
   active_nav: pagename
   ---
   ```
2. Create `css/pagename.css` for page-specific styles
3. Add the nav link in `_includes/nav.html` if needed

### Replacing images
Drop the new image into `images/` and update the `src` attribute in the relevant HTML file. Commit and push.

### Updating copy
For small text changes, edit the HTML file directly and push. For recurring content (testimonials, pricing, settings), use the CMS at `/admin`.

### Dependencies
- To update Eleventy: `npm install @11ty/eleventy@latest`
- To update Netlify Blobs: `npm install @netlify/blobs@latest`
- Always commit `package-lock.json` after updating — Netlify CI uses `npm ci` which requires it
